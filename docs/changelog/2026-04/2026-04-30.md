# Changelog 2026-04-30

- 🏷️ **Status:** [✅] Concluído
- 📅 **Data:** 2026-04-30
- 📝 **Atividade:** Correção da Tela de Login, Autenticação e Ambiente de Desenvolvimento
- 🎯 **Objetivo:** Melhorar a UX mobile no login, permitir autenticação por nome ou e-mail, e criar ambiente Docker de desenvolvimento com hot-reload.
- 📂 **Locais:** `frontend/src/main.jsx`, `frontend/src/views/public/Login.jsx`, `frontend/src/hooks/useLogin.js`, `frontend/src/context/AuthContext.jsx`, `frontend/src/services/api.js`, `frontend/vite.config.js`, `backend/src/controllers/auth.controller.js`, `backend/src/services/auth.service.js`, `backend/src/repositories/auth.repository.js`, `docker-compose.dev.yml`
- 🛠️ **Implementações:**
  - Alterada posição dos toasts de `top-right` para `bottom-center` no `main.jsx` para evitar que o aviso nativo do navegador (salvar senha) cubra mensagens de erro no mobile.
  - Adicionado método `buscarPorNomeOuEmail` no `AuthRepository` para buscar professor por nome ou e-mail.
  - Atualizado `AuthService.login` e `AuthController.login` para aceitar o campo `identificador` em vez de `nome`.
  - Renomeado campo `nome` para `identificador` no hook `useLogin` e no componente `Login.jsx`.
  - Atualizado label do campo de entrada para "Usuário ou E-mail" e placeholder para "Seu nome ou e-mail".
  - Removida qualquer validação de limite de 20 caracteres no fluxo de login (o limite permanece apenas em criação/alteração de senha).
  - Corrigido interceptor do Axios em `api.js` para não redirecionar a página quando o erro 401 ocorre na rota de login, evitando recarregamento e limpeza dos campos.
  - Campos de usuário e senha agora ficam destacados em vermelho (borda e mensagem) quando as credenciais são inválidas.
  - Ao errar as credenciais, o campo de senha é limpo e o input é recriado (via `key` dinâmica), impedindo que o navegador sugira salvar a senha incorreta. A sugestão de preenchimento para credenciais já salvas permanece ativa.
  - Criado `docker-compose.dev.yml` para ambiente de desenvolvimento com hot-reload (Vite + nodemon).
  - Ajustado `vite.config.js` para aceitar `VITE_API_URL` e expor `host: '0.0.0.0'` para funcionar dentro do container Docker.
- ✅ **Resultado:** O login agora pode ser realizado tanto com nome quanto com e-mail, os alertas de erro/sucesso aparecem na parte inferior da tela, ao errar a senha a página não recarrega mais e os campos ficam destacados em vermelho. Além disso, o ambiente de desenvolvimento Docker agora suporta hot-reload, eliminando a necessidade de recompilar a cada mudança.
