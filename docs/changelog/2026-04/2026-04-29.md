# Changelog 2026-04-29

- 🏷️ **Status:** [✅] Concluído
- 📅 **Data:** 2026-04-29
- 📝 **Atividade:** Correção de bug na edição de professores (casa e perfil de permissão)
- 🎯 **Objetivo:** Garantir que ao editar um professor, os campos `casa_id` e `permissao` sejam corretamente enviados ao backend, refletindo as alterações feitas pelo administrador.
- 📂 **Locais:** `frontend/src/hooks/useAdminProfessores.js`, `frontend/src/views/admin/AdminProfessores.jsx`
- 🛠️ **Implementações:**
  - Identificado bug de inconsistência de tipo em selects controlados do React: os valores vindos do backend (números inteiros via JSON) não correspondiam ao tipo string que o DOM HTML atribui às `<option>` dos selects.
  - Fix aplicado no hook `useAdminProfessores.js`: `permissao` e `casa_id` agora são convertidos via `String()` em `abrirEditar` ao popular o estado do formulário.
  - Padronizado `<option value="1">` e `<option value="2">` para literais string no select de permissão da view.
  - Removida conversão `Number()` redundante no `onChange` do select de permissão (a conversão final em `handleSalvar` já garante o tipo correto antes de enviar ao backend via `Number(form.permissao)`).
- ✅ **Resultado:** Os selects de casa e perfil de permissão agora refletem e enviam corretamente os valores editados pelo administrador.
