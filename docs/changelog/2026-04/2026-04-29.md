# Changelog 2026-04-29

- 🏷️ **Status:** [✅] Concluído
- 📅 **Data:** 2026-04-29
- 📝 **Atividade:** Correção de bug na edição de professores (casa e perfil de permissão)
- 🎯 **Objetivo:** Garantir que ao editar um professor, os campos `casa_id` e `permissao` sejam corretamente enviados ao backend, refletindo as alterações feitas pelo administrador.
- 📂 **Locais:** `frontend/src/views/admin/AdminProfessores.jsx`
- 🛠️ **Implementações:**
  - Identificado bug de inconsistência de tipo em selects controlados do React: os valores vindos do backend (números inteiros) não correspondiam ao tipo string que o DOM HTML atribui às `<option>` dos selects.
  - Corrigido `abrirEditar`: `permissao` e `casa_id` agora são convertidos via `String()` ao popular o estado do formulário.
  - Corrigido `abrirCriar`: `casa_id` também convertido com `String()`.
  - Padronizado `<option value="1">` e `<option value="2">` para literais string no select de permissão.
  - Padronizado `value={String(c.id)}` nas options do select de casa.
  - Removida conversão `Number()` redundante no `onChange` do select de permissão (a conversão final no `handleSalvar` já garante o tipo correto antes de enviar ao backend).
- ✅ **Resultado:** Os selects de casa e perfil de permissão agora refletem e enviam corretamente os valores editados pelo administrador.
