- 🏷️ **Status:** [✅] Concluído
- 📅 **Data:** 2026-04-20
- 📝 **Atividade:** Correção de Engine de Database Migrations e Conflict Docker
- 🎯 **Objetivo:** Resolver bug crítico onde as migrations em NodeJS não conseguiam executar statements de comentários contendo `;` ou criavam databases com nomes conflitantes, unificando a fonte da verdade para o arquivo puramente SQL (`schema.sql`).
- 📂 **Locais:** `backend/src/config/runMigrations.js`, `backend/migrations/001_create_schema.sql`, `docker-compose.yml`
- 🛠️ **Implementacoes:**
  - Substituição da lógica arcaica de `split(';')` em Node pelo uso robusto nativo da option `multipleStatements: true` do `mysql2` na função `conn.query(sql)`.
  - Remoção de statements de `CREATE DATABASE` e `USE` indevidos de dentro do arquivo transacional de migration `001_create_schema.sql`.
  - O arquivo `backend/migrations/001_create_schema.sql` foi reescrito para ser uma cópia fiel do artefato original referencial de tabelas em `sql/schema.sql`.
  - Correção do ENV default da variável `MYSQL_DATABASE` em docker-compose de `tacacasas` para `taca_das_casas` para não frustrar o backend que tentaria buscar no server um database diferente do configurado na inicialização DB.
- ✅ **Resultado:** Aplicação agora executa migrations confiantemente conectada ao driver nativo MySQL2, obedecendo regras docker e baremetal simultaneamente sem duplicação ou desajustes de banco.
