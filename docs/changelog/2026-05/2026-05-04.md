  - Backend lancamentos: repository, service e controller adaptados para campos de texto (professor, casa, aluno, turma, justificativa, complemento). Permissão por comparação de nome.

  - Perfil controller: CPF no SELECT, limpeza de dígitos do telefone antes de salvar.
  - Frontend formatters: nova função formatCPF() com máscara progressiva XXX.XXX.XXX-XX.
  - Frontend validators: nova função isValidCPF() com algoritmo completo de dígitos verificadores (verificação preliminar + cálculo dos dois dígitos).
  - Frontend hooks: adaptados para nova estrutura (filtros por texto, complemento, CPF/telefone formatados com envio cru).
  - Frontend views: AdminProfessores com campos CPF/Email/Telefone e validação. AdminCasas sem edição. AdminTurmas com nome read-only. ListagemLancamentos com campos v3 e complemento. LancarPontos com textarea de complemento. Perfil com nome/CPF read-only.
